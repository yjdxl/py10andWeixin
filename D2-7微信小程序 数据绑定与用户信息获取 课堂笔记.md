# 二、数据绑定 核心知识点

## 2.1 基本显示（数据绑定基础）

1. 页面文件（微信ML/HTML）：通过「大括号」编写模板，实现数据展示。

2. 逻辑文件（JS）：在 `data` 中定义需要展示的参数，数据会自动绑定到页面的大括号中，实现页面数据渲染。

3. 关键：无需额外操作，仅需在 `data` 中定义参数，即可完成基础数据绑定。

## 2.2 数据更新

1. 核心方法：必须使用 `setData` 方法修改数据，不能直接修改 `data` 中的参数。

2. 操作逻辑：绑定事件（如点击事件），在事件对应的函数中，通过 `setData` 修改数据，页面会自动同步更新，无需手动刷新。

3. 易错点：忘记使用 `setData`，直接修改 `data`，会导致页面数据不更新。

## 2.3 补充说明

获取微信头像、用户信息等操作，与「数据绑定」本身无依赖关系：获取信息是独立操作，数据绑定仅用于将获取到的信息展示在页面上。

# 三、用户信息获取 两种方式（重点）

## 3.1 方式一：官方不推荐（旧方式）

1. 页面写法：在页面（微信ML/HTML）中直接绑定 `getUserInfo` 相关方法（如 `bind:getUserInfo`）。

2. 逻辑写法：在JS文件中定义对应方法，调用微信提供的 `getUserInfo` 接口获取用户信息。

3. 问题：需要手动设置授权（如修改 `setting` 配置），且官方已不推荐这种绑定方式，易出现报错。

## 3.2 方式二：官方推荐（button按钮方式，重点掌握）

这是目前获取用户信息的标准方式，可自动触发授权弹窗，无需手动配置。

1. 页面写法（固定格式，不可修改）：

在页面中编写 button 按钮，必须包含固定属性：`open-type="getUserInfo"` 和 `bindgetuserinfo="自定义方法名"`。

示例：<button open-type="getUserInfo" bindgetuserinfo="fetchInfo">获取用户信息</button>

2. 逻辑写法：

（1）在JS文件中定义与 `bindgetuserinfo` 绑定的自定义方法（如示例中的`fetchInfo`）。

（2）方法内部可通过回调函数获取用户信息，无需手动调用 `getUserInfo`接口（button 点击后自动触发）。

3. 核心优势：第一次点击按钮时，会自动弹出授权弹窗，用户点击「允许」后，自动完成授权并获取用户信息，无需手动修改配置。

# 四、完整实例（登录页面获取用户信息）

## 4.1 页面设计（login页面）

1. 页面用途：单独创建 login 页面，用于用户授权登录、获取用户信息。

2. 页面内容：包含 button 按钮（用于触发授权），以及需要展示的用户信息（如用户名、头像）。

3. 数据绑定：在 JS 的 `data` 中定义默认参数（如 `name: ""`、`pass: "默认路径"`），用于展示用户信息。

## 4.2 逻辑编写（JS文件）

1. 定义自定义方法（如 `fetchInfo`），用于接收授权后的用户信息。

2. 方法内部处理：

（1）通过回调函数 `success(res)` 获取用户信息（res 中包含用户昵称、头像等数据）。

（2）使用 `setData` 将获取到的信息（如昵称 `res.userInfo.nickName`、头像 `res.userInfo.avatarUrl`）赋值给 `data` 中的参数，实现页面展示。

3. 易错点：回调函数内部的 `this` 指向问题，需在方法外部定义 `const that = this`，用 `that.setData` 修改数据（避免 `this.setData` 失效）。

# 五、注意事项（关键易错点）

1. 授权前提：小程序获取用户信息，必须经过用户授权（用户点击「允许」才能获取，点击「拒绝」则无法获取）。

2. 拒绝授权后的处理：

（1）用户第一次拒绝授权后，再次点击按钮不会再弹出授权弹窗，无法获取用户信息。

（2）解决方案：调用微信接口 `wx.openSetting()`（括号内无需填写参数），手动打开授权设置页面，重新允许小程序获取用户信息。

3. 开发规范：优先使用方式二（button 按钮方式），避免使用方式一，防止出现报错和兼容性问题。

4. 核心提醒：区分「获取用户信息」和「数据绑定」的关系——获取信息是独立操作，数据绑定仅用于展示，两者无直接依赖，但需配合使用。
