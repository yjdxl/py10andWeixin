# 微信小程序&Vue相关循环指令及案例笔记

## 一、核心前提：数据绑定

后端与前端完成数据绑定后，只需修改后端JS代码，前端页面就会同步发生变化，这是后续所有数据展示（包括循环展示）的基础。

## 二、核心指令：循环展示指令（Fore相关）

### 2.1 指令作用

用于在前端页面上，循环展示后端传递的数据（如列表、字典），避免手动写死每一条数据，提升开发效率。

### 2.2 适用场景

微信小程序、Vue.js中均有对应实现，核心逻辑一致，仅语法略有差异。

### 2.3 具体语法

- **微信小程序**：使用 `wx:for` 指令，语法为 `wx:for="{{ 后端数据名 }}"`

- **Vue.js**：使用 `v-for` 指令，语法为 `v-for="item in 后端数据名"`（与Python的for循环逻辑类似）

## 三、循环展示案例（重点复习）

### 3.1 案例1：循环展示列表数据（商品列表）

#### 步骤1：后端JS定义列表数据

假设后端定义一个商品列表数据，示例：

```javascript
// 后端JS代码
data: {
  dataList: ["白号尾", "海狗", "长鑫"] // 列表数据，可自定义内容
}

```

#### 步骤2：前端页面循环展示

核心：通过循环指令，将列表中的每一个元素渲染到页面，无需手动写死每一条商品信息。

- 微信小程序写法：使用 `wx:for="{{ dataList }}"`，默认生成两个关键变量：

    - `item`：循环的每一个元素（默认名称，可自定义）

    - `index`：循环的索引（从0开始，默认名称，可自定义）

- Vue.js写法：使用 `v-for="item in dataList"`，默认同样有 `item`（元素）和 `index`（索引）

#### 步骤3：自定义循环变量名称（可选）

若不想使用默认的 `item` 和 `index`，可自定义名称：

- 微信小程序：使用 `wx:for-item="自定义元素名"` 和 `wx:for-index="自定义索引名"`

- 示例：`wx:for="{{ dataList }}" wx:for-item="x" wx:for-index="idx"`，此时 `x` 对应元素，`idx` 对应索引

### 3.2 案例2：循环展示字典数据

#### 步骤1：后端JS定义字典数据

假设后端定义一个用户信息字典，示例：

```javascript
// 后端JS代码
data: {
  userInfo: {
    name: "Alex",
    age: 18
  } // 字典数据，key-value形式
}

```

#### 步骤2：前端页面循环展示

核心：字典循环与列表循环逻辑一致，默认变量含义有差异：

- 默认 `index`：对应字典的 **key**（如name、age）

- 默认 `item`：对应字典的 **value**（如Alex、18）

补充：也可通过自定义变量名，修改 `key` 和 `value` 的对应名称，方法同列表循环。

### 3.3 关键注意点

当后端修改列表/字典数据时，前端页面会**动态同步更新**，无需手动修改前端代码（体现数据绑定的核心优势）。

- 循环指令核心：`wx:for`（微信小程序）、`v-for`（Vue），用于循环渲染后端数据
