# 微信小程序开发环境搭建笔记

# 一、Python环境准备（2.1）

无需额外搭建Python基础环境，核心是为小程序开发创建**虚拟环境**，并在虚拟环境中安装以下必备依赖：

- jungle（必备）

- drf（Django REST framework，必备）

- pycharm（开发工具，必备）

说明：其他依赖暂不需求，按上述要求安装即可完成Python侧环境准备。

# 二、微信小程序环境搭建（2.2）

## 2.2.1 申请微信公众平台账号

1. 搜索“微信公众平台”，进入官方网站后点击“注册”。

2. 注册时选择“小程序”类型（务必选择正确，不可选错）。

3. 账号类型选择：

    - 企业号：功能最完整（含微信支付等核心功能），但需提供公司资质、法人信息，缴纳300元押金，手续繁琐。

    - 个人号：手续简单，无需资质和押金，但功能受限（如无微信支付功能），适合初学者临时使用。

4. 注册流程：按提示填写邮箱、密码等信息，完成注册后登录即可。

注意：个人号在后续学习“支付环节”时无法实操，可通过老师的企业号演示了解功能。

## 2.2.2 保存个人APPID

1. 登录微信公众平台后，点击左侧菜单栏“开发”，再选择“开发设置”。

2. 在开发设置页面中，找到“APPID”（小程序的唯一身份标识），复制并保存（建议单独记录，后续创建项目需用到）。

注意：不可使用他人的APPID，否则后续代码开发可能出现权限问题，且他人可获取你的项目信息。

## 2.2.3 下载并安装微信开发者工具

1. 在微信公众平台首页，下拉页面找到“开发者工具”入口，点击进入。

2. 根据自己的电脑系统（Windows/macOS），下载“稳定版”开发者工具。

3. 下载完成后，按默认步骤安装，安装成功后打开工具。

4. 首次打开工具时，需用注册小程序的微信账号扫码登录，登录后进入项目管理页面。

## 2.2.4 创建小程序项目

1. 登录开发者工具后，点击“+”号，创建新项目。

2. 填写项目信息：
        

    - 项目名称：自定义（如“拍卖小程序”），建议贴合项目功能。

    - 项目路径：选择电脑本地文件夹（建议单独创建一个小程序项目目录，便于管理）。

    - APPID：粘贴之前保存的个人/企业APPID（不可留空，也不可使用测试号，避免功能受限）。

    - 后端服务：选择“不使用云开发”（咱们采用本地开发模式）。

3. 点击“创建”，即可生成默认的小程序项目框架。

# 三、小程序项目目录及文件说明

## 3.1 默认项目框架说明

创建项目后，工具会自动生成一套默认文件，初学者可将这些默认文件全部删除，从零开始开发（删除后需手动创建核心文件，避免报错）。

开发者工具界面分为三栏：
预览页面：模拟手机端效果，实时查看开发成果。代码目录：展示项目所有文件，可进行文件创建、删除、编辑操作。代码编辑区：编写具体的代码（页面结构、样式、逻辑）。

## 3.2 核心文件及作用

### 3.2.1 全局文件（项目根目录下）

- app.json：全局配置文件（必选），核心配置“pages”（指定小程序所有页面路径），需为数组格式，至少包含一个页面路径。

- app.wxss：全局样式文件，所有页面均可共用此处的样式（如通用字体、颜色）。

- app.js：全局逻辑文件，可存放所有页面共用的数据和方法（如全局变量）。

注意：app.json是必选文件，若删除需重新创建，且至少包含空字典，后续添加pages配置，否则项目会报错。

### 3.2.2 页面文件（pages目录下）

每个页面需单独创建一个文件夹（如index），每个页面文件夹下默认包含4个文件，缺一不可：

- 页面.json：当前页面的单独配置文件，仅对当前页面生效（如页面标题），可留空。

- 页面.wxml：页面结构文件，相当于HTML，用于编写页面布局（仅支持微信自定义标签）。

- 页面.wxss：当前页面的单独样式文件，仅对当前页面生效，与其他页面样式不冲突。

- 页面.js：当前页面的逻辑文件，用于处理页面交互、数据渲染等。

便捷操作：在pages目录下，新建“Page”并输入页面名称（如index），工具会自动生成该页面的4个文件，且自动在app.json中添加该页面路径。

### 3.2.3 其他文件

- utils：工具类文件夹，存放公共工具方法（如分页、通用函数），可删除，按需创建。

- sitemap.json：微信爬虫配置文件，用于控制微信爬虫抓取小程序页面（影响小程序搜索权重），无强制要求，可删除。

# 四、基础开发入门（简化版）

## 4.1 核心页面标签（初学者必记）

微信小程序不使用HTML标签，使用自定义标签，初期记住2个即可：

- text标签：相当于HTML的span标签，用于显示文本内容。

- view标签：相当于HTML的div标签，用于布局（包裹其他标签）。

## 4.2 样式编写

在页面.wxss文件中编写样式，语法与CSS基本一致：

1. 给标签添加class属性（如class="c1"）。

2. 在.wxss文件中编写对应样式（如.c1 { color: red; }），保存后预览页面即可看到效果。

## 4.3 预览与真机调试

- 预览：点击开发者工具顶部“预览”按钮，生成二维码，用微信扫码即可在手机上查看页面效果（需确保app.js文件存在，否则会报错）。

- 真机调试：点击“真机调试”，扫码后手机会同步显示页面，可实时查看调试信息（如console.log输出），适合复杂项目调试，初期暂用不到。

注意：项目文件过大会导致预览失败，开发初期可在工具中查看效果，无需频繁预览。

- 默认生成的文件可删除，从零开发更便于理解页面结构，但需确保app.json文件存在且配置正确。
