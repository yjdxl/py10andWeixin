# 微信小程序图片上传功能开发笔记（课堂整理）

# 一、页面结构（WXML）编写

## 1. 核心结构设计

页面主要包含两部分：图片上传提示、图片预览容器，核心标签及作用如下：

- view标签1：用于提示用户上传图片，内容为“请上传图片”，绑定点击事件（后续用于触发图片选择）。

- view标签2：添加class="container"，作为图片预览的容器，用于包裹所有选中的图片。

- image标签：循环渲染图片列表（imageList），src属性绑定循环的每个元素item（即图片路径），实现图片预览。

## 2. 图片样式设置（基础）

为container容器下的所有image标签设置样式，避免图片过大、无间距，样式如下：

- 宽度：200rpx（适配小程序屏幕）

- 高度：200rpx（与宽度一致，保持图片比例）

- 内边距（padding）：5px（给图片添加小边距，避免堆叠拥挤）

默认情况下，图片会垂直堆叠展示，后续通过选择图片动态添加预览。

# 二、逻辑功能（JS）编写

## 1. 核心需求

点击“请上传图片”，弹出弹窗让用户选择图片（相册/相机），选择完成后，在页面容器中预览图片；支持多次选择，新增图片追加到预览列表（不覆盖原有图片）。

## 2. 步骤1：绑定点击事件

给“请上传图片”对应的view标签绑定tap事件，事件名定义为uploadImage，用于触发图片选择功能。

## 3. 步骤2：调用微信内置API（wx.chooseImage）

微信小程序中，操作相册、相机等功能，需使用微信内置API，图片选择对应的API为wx.chooseImage，核心参数及说明如下：

- count：最多可选择的图片数量，默认9张（与朋友圈图片限制一致），示例：count: 9。

- sizeType：图片尺寸类型，可选值为['original', 'compressed']（original=原图，compressed=压缩图），两者都添加可支持用户选择原图/压缩图。

- sourceType：图片来源，可选值为['album', 'camera']（album=相册，camera=相机），两者都添加可支持用户从相册选择或现场拍摄。

- success：选择图片成功后的回调函数，参数res中包含选择的图片信息，是核心回调。

- fail（可选）：选择图片失败后的回调函数，用于处理异常。

- complete（可选）：无论选择成功/失败，都会执行的回调函数（类似Python的finally），简化开发时可省略。

调用示例核心代码逻辑：wx.chooseImage({ 参数配置 })。

## 4. 步骤3：处理选择结果（success回调）

选择图片成功后，success回调函数中的res参数会返回两个关键数组，用于获取图片路径：

- tempFilePaths：图片临时路径数组，元素为图片的本地临时地址（可直接用于image标签的src预览），用法简单，适合基础预览。

- tempFiles：图片详细信息数组，元素为字典，包含图片路径、大小等信息，适合需要限制图片大小的场景。

基础开发中，使用tempFilePaths即可满足预览需求，获取方式：res.tempFilePaths。

## 5. 步骤4：更新页面预览（setData）

小程序中，页面数据更新必须通过setData方法，无法直接修改数据后自动渲染，核心逻辑如下：

- 定义页面初始数据imageList（空数组），用于存储选中的图片路径。

- 选择图片成功后，将res.tempFilePaths赋值给imageList，通过setData更新数据，页面会自动循环渲染图片。

- 核心代码：that.setData({ imageList: res.tempFilePaths })（that = this，避免回调函数中this指向异常）。

## 6. 步骤5：实现图片追加（不覆盖原有图片）

默认情况下，直接赋值会覆盖原有imageList，需实现“追加”效果，核心思路的是拼接两个数组（原有图片列表 + 新选择图片列表）：

- JS中数组拼接使用concat方法（类似Python的extend），语法：原有数组.concat(新数组)，拼接后生成新数组，不修改原数组。

- 核心代码：that.setData({ imageList: that.data.imageList.concat(res.tempFilePaths) })。

- 效果：新选择的图片会追加到原有预览列表末尾，不会覆盖之前选中的图片。

# 三、关键注意事项

- 临时路径特性：选择图片后获取的tempFilePaths是微信小程序的内存临时路径，页面刷新后会丢失，仅用于前端预览，无法长期保存。

- 企业级开发要求：实际项目中，需将选中的图片通过后端接口上传到服务器，获取服务器上的图片永久地址，再渲染到页面，确保页面刷新后图片仍存在。

- API扩展：微信小程序的媒体API除了chooseImage（获取图片），还有获取视频、位置等功能，按需调用即可，无需提前掌握所有API。

- setData强制使用：页面数据更新必须通过setData，直接修改this.data中的数据，页面不会自动渲染。

# 四、核心代码总结

## 1. WXML核心代码（简化版）

```xml
<!-- 图片上传提示，绑定点击事件 -->
<view bindtap="uploadImage">请上传图片</view>
<!-- 图片预览容器 -->
<view class="container">
  <!-- 循环渲染图片列表 -->
  <image wx:for="{{imageList}}" wx:key="index" src="{{item}}"></image>
</view>
```

## 2. JS核心代码（简化版）

```javascript
Page({
  // 初始数据：图片列表
  data: {
    imageList: []
  },
  // 图片上传点击事件
  uploadImage() {
    const that = this;
    // 调用微信选择图片API
    wx.chooseImage({
      count: 9, // 最多选择9张
      sizeType: ['original', 'compressed'], // 支持原图、压缩图
      sourceType: ['album', 'camera'], // 支持相册、相机
      success(res) {
        // 拼接数组，实现图片追加（不覆盖）
        const newImageList = that.data.imageList.concat(res.tempFilePaths);
        // 更新页面数据，渲染图片
        that.setData({
          imageList: newImageList
        });
      }
    })
  }
})
```
