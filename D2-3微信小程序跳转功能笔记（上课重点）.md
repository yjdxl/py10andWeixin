# 微信小程序跳转功能笔记（上课重点）

# 一、核心需求：页面跳转实现

场景：点击页面中的某个标签（如view、图片等），跳转到小程序的其他页面，可携带参数并在目标页面接收。

前提：已创建基础小程序项目，初始开发页面为Index页面，新建目标跳转页面（示例中为Redirect页面，实际可自定义，如detail页面）。

# 二、核心知识点拆解

## 2.1 给标签绑定点击事件

核心：给需要点击跳转的标签（如view）绑定点击事件，触发对应JS函数。

1. 标签绑定事件：在标签上添加 `bindtap` 属性（绑定点击事件的核心属性），指定事件函数名。

2. 示例（Index页面wxml）：

3. `<view bindtap="clickMe">点我跳转</view>`（函数名自定义，此处为clickMe）

4. 编写事件函数：在当前页面的JS文件（如Index.js）中，在Page配置对象内定义对应函数。

5. 示例（Index.js）：

6. `Page({
  // 内置函数（默认生成，无需修改，暂时忽略）
  data: {},
  onLoad(options) {},
  onReady() {},
  onShow() {},
  onHide() {},
  onUnload() {},
  // 自定义点击事件函数
  clickMe: function() {
    // 函数逻辑（跳转、打印调试等）
    console.log("点我了"); // 调试用，可查看函数是否触发
  }
})`

## 2.2 给点击事件传递参数

核心：通过 `data-*` 前缀给标签添加属性，传递参数，在事件函数中通过事件对象e获取。

1. 标签添加参数：以 `data-参数名` 形式添加属性，赋值为需要传递的参数。

2. 示例（Index页面wxml）：

3. `<view bindtap="clickMe" data-nid="123" data-name="测试">点我跳转</view>`

4. 事件函数接收参数：在clickMe函数中添加参数e（事件对象），通过 `e.currentTarget.dataset` 获取所有传递的参数。

5. 示例（Index.js）：

6. `clickMe: function(e) {
  // 打印事件对象，查看参数位置（调试用）
  console.log(e);
  // 获取传递的参数
  let nid = e.currentTarget.dataset.nid; // 获取data-nid的值（123）
  let name = e.currentTarget.dataset.name; // 获取data-name的值（测试）
}`

7. 注意：所有通过data-*传递的参数，都会存放在 `e.currentTarget.dataset` 中，参数名与data-后的名称一致。

## 2.3 页面跳转（核心方法）

核心：使用微信小程序内置API `wx.navigateTo` 实现页面跳转，可拼接URL参数传递数据。

1. 跳转API格式：

2. `wx.navigateTo({
  url: "跳转目标页面的路径?参数名1=参数值1&参数名2=参数值2" // 路径+参数拼接
})`

3. 路径要求：与app.json中注册的页面路径一致，格式为 `pages/页面文件夹/页面名称`（如pages/Redirect/Redirect）。

4. 带参数跳转示例（结合前面的参数传递）：

5. `clickMe: function(e) {
  let nid = e.currentTarget.dataset.nid;
  // 拼接URL，传递nid参数
  wx.navigateTo({
    url: "/pages/Redirect/Redirect?id=" + nid
  });
}`

6. 效果：点击标签后，自动跳转到Redirect页面，并携带id参数（值为123）。

## 2.4 目标页面接收跳转参数

核心：在目标页面（如Redirect.js、detail.js）的 `onLoad` 生命周期函数中，通过options参数接收跳转传递的参数。

1. onLoad函数说明：页面加载时自动触发，是接收跳转参数的唯一入口，options参数中包含所有拼接在URL上的参数。

2. 示例（Redirect.js）：

3. `Page({
  // 页面加载时触发，接收跳转参数
  onLoad(options) {
    // 打印options，查看接收的参数（调试用）
    console.log(options);
    // 获取跳转传递的id参数
    let id = options.id; // 此处id与URL中拼接的参数名一致
    console.log("接收的id：", id); // 输出：接收的id：123
  }
})`

4. 注意：options中的参数名，必须与跳转时URL拼接的参数名完全一致。

# 三、重点总结

1. 跳转功能核心两步：① 给标签绑定bindtap点击事件 + 编写事件函数；② 事件函数中调用wx.navigateTo API实现跳转。

2. 参数传递两种方式：
        

    - 标签传递参数：data-* 属性，通过e.currentTarget.dataset获取；

    - 跳转传递参数：URL拼接（?参数名=值&参数名=值），通过目标页面onLoad的options获取。

3. onLoad生命周期函数：仅在页面加载时触发一次，专门用于接收跳转参数，不可省略。
